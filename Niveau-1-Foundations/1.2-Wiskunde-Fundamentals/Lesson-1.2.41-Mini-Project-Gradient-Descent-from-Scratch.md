# Lesson 1.2.41: Mini-Project - Gradient Descent from Scratch

**Estimated Time**: 8 hours | **Difficulty**: Intermediate  
**Prerequisites**: Lessons 1.2.26 - 1.2.40 | **Topic**: Calculus for ML Integration

---

## Learning Objectives

By the end of this lesson, you will be able to:
- Implement gradient descent for a simple loss
- Track loss over iterations
- Choose a learning rate
- Explain convergence behavior

---

## Project Overview

Build a small optimizer for f(x)=x^2 and extend to a 2D function.

---

## Exercises

### Difficulty 1: Starter
- Exercise 1.2.41.1: Write gradient for f(x)=x^2.
- Exercise 1.2.41.2: Implement one update step.
- Exercise 1.2.41.3: Print loss each step.

### Difficulty 2: Intermediate
- Exercise 1.2.41.4: Implement gradient descent loop.
- Exercise 1.2.41.5: Try two learning rates.
- Exercise 1.2.41.6: Add a stopping rule.

### Difficulty 3: Advanced
- Exercise 1.2.41.7: Extend to f(x,y)=x^2+y^2.
- Exercise 1.2.41.8: Add momentum term.
- Exercise 1.2.41.9: Plot the loss curve.

---

## Mini-Project: Optimizer

Implement gradient descent and report iterations and final x.

---

## Quiz & Assessment

### Quick Check (True/False)
1. Gradient descent requires gradients. (True)
2. Learning rate does not affect convergence. (False)
3. Loss should decrease over time. (True)
4. Stopping rules prevent infinite loops. (True)

### Conceptual Questions
5. Why does gradient descent converge?
6. When does it diverge?
7. What is a practical stopping rule?

### Coding Challenge
8. Implement gradient descent for f(x)=x^2.

---

## Key Takeaways

- Gradient descent is simple but powerful
- Learning rate is critical
- Loss tracking shows progress
- Stopping rules prevent wasted steps
